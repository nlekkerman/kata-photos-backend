## Model Fields Added

`FieldNote` model in `gallery/models.py`:

| Field | Type | Constraints |
|---|---|---|
| `slug` | `SlugField(max_length=200)` | `unique=True` |
| `title_en` | `CharField(max_length=200)` | Required |
| `title_bs` | `CharField(max_length=200)` | `blank=True` |
| `excerpt_en` | `TextField` | `blank=True` |
| `excerpt_bs` | `TextField` | `blank=True` |
| `body_en` | `TextField` | Required |
| `body_bs` | `TextField` | `blank=True` |
| `location` | `CharField(max_length=200)` | `blank=True` |
| `is_published` | `BooleanField` | `default=False` |
| `published_at` | `DateTimeField` | `null=True, blank=True` |
| `cover_image` | `ForeignKey('MediaItem')` | `null=True, blank=True, on_delete=SET_NULL` |
| `created_at` | `DateTimeField` | `auto_now_add=True` |
| `updated_at` | `DateTimeField` | `auto_now=True` |

`Meta.ordering = ['-published_at', '-created_at']`

**Model-level validation** (`clean()` method):

- If `is_published=True` and `title_en` is empty → `ValidationError` on `title_en`
- If `is_published=True` and `body_en` is empty → `ValidationError` on `body_en`

Django's `clean()` is called during form validation (Django admin `save_model`, `ModelForm`). It is not called on direct `QuerySet.save()` calls — this is consistent with Django conventions and sufficient for admin-managed content.

## Public Endpoints Added

Both endpoints are mounted under the existing `/api/gallery/` prefix (no change to `config/urls.py`).

| Method | URL | View | Serializer |
|---|---|---|---|
| `GET` | `/api/gallery/field-notes/` | `FieldNoteListView` | `FieldNoteListSerializer` |
| `GET` | `/api/gallery/field-notes/<slug>/` | `FieldNoteDetailView` | `FieldNoteDetailSerializer` |

Both endpoints support `?lang=en` and `?lang=bs` via the existing `LangContextMixin`.

## Edge Cases Verified

| # | Scenario | Result |
|---|---|---|
| 1 | No field notes exist | `GET /api/gallery/field-notes/?lang=en` → `[]` HTTP 200 ✓ |
| 2 | Only unpublished notes exist | After setting `is_published=False` → `[]` HTTP 200 ✓ |
| 3 | One published note exists | List returns array with 1 item ✓ |
| 4 | Multiple published notes | `Meta.ordering = ['-published_at', '-created_at']` — ordering verified by model definition |
| 5 | Invalid slug | `GET /api/gallery/field-notes/nonexistent-slug/?lang=en` → HTTP 404 `{"detail":"No FieldNote matches the given query."}` ✓ |
| 6 | Unsupported `?lang=fr` | `LangContextMixin` falls back to `'en'`; response returns English translations ✓ |
| 7 | Missing Bosnian translation (`body_bs=""`) | `resolve_translated` returns `body_en`; no empty string surfaced ✓ |
| 8 | Empty excerpt | `excerpt_en` and `excerpt_bs` are `blank=True`; empty string returned; no crash ✓ |
| 9 | Missing cover image | `"cover_image": null` in response ✓ |
| 10 | Draft notes not appearing | After `is_published=False` update, public list returns `[]` ✓ |
| 11 | Published note with `published_at=None` | No crash; `"published_at": null` in response; sorts to end of default ordering |
| 12 | Cover image URL is backend-provided | `FieldNoteCoverSerializer.get_thumbnail_url` uses `_get_thumbnail_url(obj, request)` → `request.build_absolute_uri()` ✓ |
| 13 | No frontend URL construction needed | Full absolute URLs returned in all URL fields ✓ |
| 14 | Public API does not expose `_en`/`_bs` fields | Confirmed: no `title_en`, `title_bs`, `excerpt_en`, `excerpt_bs`, `body_en`, `body_bs` in any response ✓ |
