## Serializer Changes

### New helper — `resolve_translated`

```python
def resolve_translated(obj, field_name, lang):
    value = getattr(obj, f"{field_name}_{lang}", "")
    fallback = getattr(obj, f"{field_name}_en", "")
    return value or fallback
```

Reads `<field>_<lang>` first. Falls back to `<field>_en` if the value is blank or the attribute does not exist. Returns an empty string only if both are blank.

## View / Language Context Logic

### `LangContextMixin`

Added to `gallery/views.py`:

```python
_ALLOWED_LANGS = ('en', 'bs')

class LangContextMixin:
    def get_serializer_context(self):
        context = super().get_serializer_context()
        raw_lang = self.request.query_params.get('lang', 'en').lower()
        context['lang'] = raw_lang if raw_lang in _ALLOWED_LANGS else 'en'
        return context
```

- `?lang=` absent → `'en'`
- `?lang=EN` → normalised to `'en'` via `.lower()`
- `?lang=fr` → not in `_ALLOWED_LANGS` → falls back to `'en'`
- `?lang=bs` → `'bs'`

Applied to all four views via multiple inheritance:

```python
class AlbumListView(LangContextMixin, generics.ListAPIView): ...
class AlbumDetailView(LangContextMixin, generics.RetrieveAPIView): ...
class AlbumMediaListView(LangContextMixin, generics.ListAPIView): ...
class MediaItemDetailView(LangContextMixin, generics.RetrieveAPIView): ...
```

`lang` is resolved once per request in `get_serializer_context` and propagated automatically to all nested serializers (DRF passes context to nested serializers).
