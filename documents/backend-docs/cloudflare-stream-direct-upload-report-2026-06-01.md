## Service Behavior — `gallery/services/cloudflare_stream.py`

- Function: `create_direct_upload(*, account_id, api_token, max_duration_seconds, expiry_seconds, creator="", meta=None) -> dict`
- Calls `POST https://api.cloudflare.com/client/v4/accounts/{account_id}/stream/direct_upload`
- Request body: JSON with `maxDurationSeconds`, `expiry` (ISO 8601 UTC), optional `creator`/`meta`
- Uses stdlib `urllib.request` — no `requests` dependency added (consistent with Phase 8 Images service)
- Timeout: 30 seconds
- Bearer token authorization header
- Returns `{"uid": "...", "upload_url": "..."}`
- Custom exception: `CloudflareStreamUploadError` — token is never included in error messages
- Errors raised on: missing config, HTTP error, network/OS error, Cloudflare `success: false`

## Endpoint Behavior

```
POST /api/gallery/videos/direct-upload/
```

**Request body:**
```json
{
  "album": 1,
  "title_bs": "Video title",
  "title_en": "",
  "description_bs": "",
  "description_en": "",
  "max_duration_seconds": 300
}
```
- `album`: optional (nullable FK)
- `title_bs`: required
- `max_duration_seconds`: optional, default 300, must be positive integer

**Successful response (201):**
```json
{
  "video": {
    "id": 1,
    "album": 1,
    "title_bs": "Video title",
    "title_en": "",
    "description_bs": "",
    "description_en": "",
    "cloudflare_uid": "abc123...",
    "cloudflare_thumbnail_url": "",
    "cloudflare_playback_url": "",
    "duration_seconds": null,
    "status": "uploading",
    "is_public": false,
    "created_at": "...",
    "updated_at": "..."
  },
  "upload_url": "https://upload.videodelivery.net/..."
}
```

**Error responses:**
- `401` / `403` — unauthenticated or non-staff user
- `400` — invalid request body (DRF validation error)
- `500` with `stream_not_configured` code — missing `CLOUDFLARE_ACCOUNT_ID` or `CLOUDFLARE_STREAM_API_TOKEN`
- `502` (`cloudflare_upload_error`) — Cloudflare API failure (follows existing Images pattern via `CloudflareServiceError`)

## Frontend Contract — Next Phase

To initiate a video upload from the browser:

1. **POST** `/api/gallery/videos/direct-upload/` with session cookie (staff only)
2. Receive `upload_url` and `video.cloudflare_uid`
3. Use the TUS protocol to upload the raw video file directly to Cloudflare using `upload_url`
   - Browser libraries: `tus-js-client` or `@uppy/tus`
   - The URL is single-use and expires after `CLOUDFLARE_STREAM_DIRECT_UPLOAD_EXPIRY_SECONDS`
4. After upload completes, Cloudflare processes the video asynchronously
5. `video.status` will remain `uploading` until a future webhook or polling mechanism updates it

Fields available immediately after creation: `id`, `cloudflare_uid`, `status: "uploading"`.
Fields populated by future status sync: `cloudflare_thumbnail_url`, `cloudflare_playback_url`, `duration_seconds`, `status: "ready"`.
